# Настройка Twenty CRM по постановке Vkusno Polezno Astana

Документ связывает **постановку задачи** ([📄 Постановка задачи twenty crm.md](../📄%20Постановка%20задачи%20twenty%20crm.md) в корне проекта) с шагами настройки Twenty CRM и интеграцией с сайтом.

---

## 1. Цель и развёртывание Twenty

- **Цель:** центральная система для заказов, клиентов, меню, производства, оплат, доставки, обратной связи (вместо хаоса в WhatsApp/Instagram).
- **Развёртывание:** [Self-hosting Twenty](https://docs.twenty.com/developers/self-hosting/docker-compose). После установки вход в CRM — по URL вашего сервера (логин/регистрация в интерфейсе Twenty).

---

## 2. Сущности и структура данных

### 2.1 Клиент (Contact)

В Twenty используется стандартный объект **Person** (Люди). Настройте поля:

| Постановка | В Twenty |
|------------|----------|
| ID | Есть по умолчанию |
| Имя | Стандартное поле `name` (firstName, lastName) |
| Телефон | Добавить поле **Phone** (или использовать стандартное, если есть) |
| WhatsApp | Кастомное поле или тот же телефон |
| Instagram username | Кастомное поле (текст) |
| Источник | Кастомное поле (select: Instagram / TikTok / рекомендация / сайт) |
| История заказов | Связь Person → Opportunities (сделки/заказы) |
| Общая сумма заказов | Вычисляемое поле или отчёт |
| Статус клиента | Кастомное поле (select: Новый / Постоянный / Корпоративный / VIP) |

**Где настраивать:** Settings → Objects → Person → добавить кастомные поля.

---

### 2.2 Меню / Продукты (Products)

Создать **кастомный объект** (Custom Object) **Product** (или «Продукт»):

| Поле | Тип |
|------|-----|
| Название | Text |
| Категория | Select: Торты / Десерты / ПП десерты / Хлеб / Пироги |
| Описание | Text (длинный) |
| Фото | Media / URL |
| Цена за единицу | Number (Currency) |
| Себестоимость | Number (внутреннее, скрыть от роли «Менеджер» при необходимости) |
| Время производства (часы) | Number |

**Где:** Settings → Objects → Create new object → Product.

Категории можно вынести в отдельный справочник или оставить как Select.

---

### 2.3 Заказ (Order)

Использовать стандартный объект **Opportunity** (Сделка) как заказ. Либо создать кастомный объект **Order** — тогда ниже адаптировать под его поля.

**Рекомендация:** использовать **Opportunity** и настроить для него pipeline (см. п. 4).

Поля Opportunity (настроить/дополнить в Twenty):

| Постановка | Поле в Twenty |
|------------|----------------|
| ID заказа | Стандартный id |
| Клиент | Связь Opportunity → Person |
| Дата заказа | Date (createdAt или кастомное) |
| Дата выдачи | Date |
| Тип получения | Select: Самовывоз / Доставка |
| Адрес доставки | Text (показывать, если «Доставка») |
| Итоговая сумма | Amount / Number |
| Предоплата | Number |
| Остаток | Number или формула |
| Статус оплаты | Select: Не оплачено / Предоплата получена / Оплачено полностью |
| Метод оплаты | Select: Kaspi / HalykBank / Freedom / Перевод / Наличные |
| Статус доставки | Select: Ожидает доставки / Передан курьеру / Доставлен / Получен заказчиком (если «Доставка») |

Состав заказа (Order Items) — отдельный объект (см. ниже).

---

### 2.4 Состав заказа (Order Items)

Создать кастомный объект **Order Item** (позиция заказа):

| Поле | Тип |
|------|-----|
| Заказ | Relation → Opportunity (или Order) |
| Продукт | Relation → Product |
| Количество | Number |
| Цена за единицу | Number (можно подтягивать из Product) |
| Сумма | Number или формула (количество × цена) |

В настройках Opportunity можно добавить связь «один ко многим» с Order Item и отображать итоговую сумму заказа (полем или через формулу/агрегат, если Twenty поддерживает).

---

### 2.5 Обратная связь (Feedback)

Создать кастомный объект **Feedback**:

| Поле | Тип |
|------|-----|
| Клиент | Relation → Person |
| Заказ | Relation → Opportunity |
| Оценка | Number (1–5) или Select |
| Комментарий | Text |

## 3. Pipeline заказа (статусы)

Настроить **Pipeline** для Opportunity (или Order) со стадиями по постановке:

1. **Новый заказ** (клиент написал)  
2. **Подтверждён** (менеджер подтвердил)  
3. **Ожидание предоплаты**  
4. **Предоплата получена**  
5. **В производстве**  
6. **Готов**  
7. **Передан на доставку** / **Готов к самовывозу**  
8. **Завершён**  
9. **Закрыт**  

**Где:** Settings → Objects → Opportunity → Pipeline stages (или аналогичный раздел для Pipelines).

Заявки с сайта при создании в CRM должны попадать в стадию **«Новый заказ»**.

---

## 4. Рабочее место менеджера

- **Главный экран:** виджеты/фильтры: новые заказы, на производство, на выдачу (по дате выдачи и стадии).
- **Раздел «Заказы»:** таблица Opportunity с колонками: ID, наименование/комментарий, доп. оформление, вес, клиент, сумма, предоплата, статус, дата выдачи.
- **Раздел «Клиенты»:** список Person, при открытии — история заказов (связанные Opportunity).
- **Раздел «Меню»:** объект Product, цены, категории.
- **Раздел «Производство»:** тот же список заказов с фильтром по дате изготовления/выдачи (сегодня, завтра, неделя).

Всё настраивается через стандартные списки, фильтры и представления (Table/Kanban) в Twenty.

## 5. Роли пользователей

| Роль постановки | В Twenty | Права |
|-----------------|----------|--------|
| Менеджер продаж | Создать роль «Sales» | Создание/редактирование заказов (Opportunity), смена стадий, работа с клиентами (Person). |
| Производство | Создать роль «Production» | Просмотр заказов, смена стадии «В производстве» и связанных. |
| Владелец | Admin / Owner | Полный доступ ко всем объектам и отчётам. |

**Где:** Settings → Workspace → Roles & Permissions (или аналогичный раздел).

## 6. Автоматизации (по возможности Twenty)

- **При создании заказа:** создавать запись клиента (Person), если такого ещё нет — это делает наш backend при отправке заявки с сайта (см. ниже).
- **При смене статуса «Готов»:** уведомление менеджеру — настроить через Automation/Workflow в Twenty (если есть).
- **При получении предоплаты:** смена статуса оплаты/заказа — через автоматизацию или ручное изменение поля.
- **Через 24 часа после «Завершён»:** задача «Запросить отзыв» — создать задачу (Task) или напоминание, если Twenty поддерживает отложенные действия.

Конкретные шаги зависят от версии Twenty и наличия модуля Automations.

## 7. Отчёты

Использовать встроенные представления и фильтры Twenty:

- Выручка за день / неделю / месяц — сумма по закрытым Opportunity за период.
- Количество заказов, средний чек — группировка и агрегаты по Opportunity.
- Популярные продукты — по связям Order Item → Product.

При необходимости экспорт в CSV/Excel для расчётов в таблицах.

## 8. Интеграция сайта с Twenty

Текущее поведение backend (папка `backend/`):

1. Заявка с формы на сайте (`order.html`) уходит в **POST /api/orders**.
2. Запись сохраняется в **PostgreSQL** (таблица `orders`).
3. Если в `.env` заданы `TWENTY_API_URL` и `TWENTY_API_KEY`, backend дополнительно:
   - создаёт **Person** (клиент: имя, телефон, комментарий/категория заказа);
   - при расширении (см. ниже) может создавать **Opportunity** в стадии «Новый заказ».

Чтобы заявки с сайта попадали в CRM как заказы в стадии «Новый заказ»:

1. В Twenty создать pipeline для Opportunity со стадией **«Новый заказ»** и запомнить её **id** (или slug) в API.
2. В `backend/server.js` в функции `syncOrderToTwenty` (или отдельной) добавить вызов создания **Opportunity**: связать с созданным Person, указать pipeline stage = «Новый заказ», при необходимости передать комментарий/категорию в поле заказа.
3. Точные имена полей и endpoint’ы (например `POST /rest/opportunities`) смотреть в Twenty: **Settings → APIs & Webhooks** → API Playground. При других именах полей в workspace поправить `syncOrderToTwenty` в `backend/server.js`.

Переменные окружения backend:

- `DATABASE_URL` — PostgreSQL (обязательно).
- `TWENTY_API_URL` — URL вашего Twenty (без слэша в конце).
- `TWENTY_API_KEY` — API-ключ из Twenty (Settings → APIs & Webhooks).
- `TWENTY_PIPELINE_STAGE_ID` — (опционально) id стадии pipeline «Новый заказ» для создания Opportunity при заявке с сайта.
