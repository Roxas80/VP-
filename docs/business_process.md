# Бизнес-процесс продаж vkusno_polezno_astana

## Участники

1. **Клиент** — делает заказ через Telegram-бот
2. **Кондитер** — принимает заказы, готовит, управляет статусами
3. **Курьер** — доставляет заказы
4. **Бухгалтер/Админ** — финансовый контроль, отчёты

## Процесс заказа

### 1. Клиент делает заказ
- Выбирает товары из каталога
- Добавляет в корзину
- Указывает адрес и телефон
- Выбирает способ оплаты:
  - **Kaspi Pay** (перевод)
  - **Halyk Bank** (перевод)
  - **Freedom Finance** (перевод)
  - **Наличные** (при получении)
  - **50% предоплата** (на будущее)

### 2. Заказ поступает кондитеру
- Уведомление с деталями заказа
- Кнопки действий:
  - ✅ Подтвердить заказ
  - ❌ Отклонить заказ
  - ⏰ Отложить (указать причину)

### 3. Подтверждение и оплата
- Клиент получает подтверждение
- Отправляется запрос на оплату
- Клиент присылает скриншот чека
- Кондитер отмечает статус оплаты:
  - 💳 Оплачено
  - ⏳ Ожидает оплату

### 4. Приготовление
- Кондитер меняет статус:
  - 👨‍🍳 Готовится
  - ✅ Готово к доставке

### 5. Назначение курьера
- Кондитер назначает курьера
- Курьер получает уведомление:
  - Адрес клиента
  - Состав заказа
  - Контакт клиента
  - Сумма (наличные/оплачено)

### 6. Доставка
- Курьер меняет статусы:
  - 🚗 В пути к клиенту
  - 📍 Прибыл
  - ✅ Доставлено
- Клиент видит статус в реальном времени

### 7. Завершение
- Клиент получает заказ
- Отзыв (опционально)
- Заказ архивируется

## Статусы заказа

| Статус | Описание | Кто меняет |
|--------|----------|------------|
| 🆕 Новый | Заказ создан | Система |
| ✅ Подтверждён | Кондитер принял заказ | Кондитер |
| ⏳ Ожидает оплату | Отправлен запрос на оплату | Система |
| 💳 Оплачено | Оплата подтверждена | Кондитер |
| 👨‍🍳 Готовится | Торт в производстве | Кондитер |
| 📦 Готово | Готово к доставке | Кондитер |
| 🚗 Назначен курьер | Курьер получил заказ | Кондитер |
| 🚚 В пути | Курьер едет к клиенту | Курьер |
| 📍 Прибыл | Курьер на месте | Курьер |
| ✅ Доставлено | Клиент получил заказ | Курьер |
| ⭐ Завершён | Заказ закрыт | Система |
| ❌ Отменён | Заказ отменён | Кондитер/Клиент |

## Роли и права

### Клиент
- Просмотр каталога
- Добавление в корзину
- Оформление заказа
- Просмотр статуса своих заказов
- Получение уведомлений

### Кондитер
- Все права клиента
- Подтверждение/отклонение заказов
- Управление статусами заказа
- Назначение курьеров
- Отметка оплаты
- Команда `/orders` — список всех заказов

### Курьер
- Получение заказов на доставку
- Изменение статуса доставки
- Связь с клиентом

### Админ
- Все права кондитера
- Доступ к статистике
- Управление каталогом
- Финансовые отчёты
