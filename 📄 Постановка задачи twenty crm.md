📄 Постановка задачи

Внедрение CRM Twenty для бренда Vkusno Polezno Astana

1\. Цель внедрения CRM



Внедрить Twenty CRM как центральную систему управления:

* заказами
* клиентами
* меню и ценами
* производством
* оплатами
* доставкой
* обратной связью
* CRM должна заменить хаотичное управление заказами через WhatsApp и Instagram.



2\. Основные бизнес-процессы, которые должна поддерживать CRM



CRM должна поддерживать полный цикл:

* Лид →
* Заказ →
* Подтверждение →
* Предоплата →
* Производство →
* Готовность →
* Доставка / самовывоз →
* Закрытие заказа →
* Обратная связь



3\. Структура данных CRM

3.1 Сущность: Клиент (Contact)



Поля:



Обязательные:

* ID
* Имя
* Телефон
* WhatsApp
* Instagram username



Дополнительные:

* Источник (Instagram / TikTok / рекомендация)
* История заказов
* Общая сумма заказов



Статус клиента:

* Новый
* Постоянный
* Корпоративный
* VIP



3.2 Сущность: Меню / Продукты (Products)



Создать каталог продуктов.



Пример структуры:

Категории:

* Торты
* Десерты
* ПП десерты
* Хлеб
* Пироги



Поля продукта:

* ID
* Название
* Категория
* Описание
* Фото
* Цена за единицу
* Себестоимость (внутреннее поле)
* Время производства (часы)



Пример:

Торт "Медовик ПП"

Цена: 12000 тг

Время изготовления: 24 часа



3.3 Сущность: Заказ (Order / Opportunity)

Ключевая сущность CRM.

Поля:

Основные:

* ID заказа
* Клиент
* Дата заказа
* Дата выдачи
* Тип получения:
* Самовывоз
* Доставка
* Адрес доставки (если есть)



3.4 Структура состава заказа



CRM должна поддерживать:

* Order → Order Items
* Каждый Order Item:
* продукт
* количество
* цена
* сумма



CRM автоматически считает:



Итоговую сумму заказа



3.5 Оплата



Поля:

* Итоговая сумма
* Предоплата
* Остаток



Статус оплаты:



Статусы:

* Не оплачено
* Предоплата получена
* Оплачено полностью



Метод оплаты:

* Kaspi
* HalykBank
* Freedom
* Перевод
* Наличные



4\. Статусы заказа (основная логика CRM)



Создать pipeline:



Статус 1: Новый заказ

(клиент написал)

↓

Статус 2: Подтвержден

(менеджер подтвердил)

↓

Статус 3: Ожидание предоплаты

↓

Статус 4: Предоплата получена

↓

Статус 5: В производстве

↓

Статус 6: Готов

↓

Статус 7: Передан на доставку

или

Готов к самовывозу

↓

Статус 8: Завершен

↓

Статус 9: Закрыт

Список заказов:

по дате выдачи

7\. Логика предоплаты

например:

Предоплата: 50% = 10000 тг



CRM показывает:

Предоплата получена

9\. Автоматизации



CRM должна автоматически:



При создании заказа:



создать запись клиента (если новый)



При смене статуса:



например:



"Готов"



CRM уведомляет менеджера



При предоплате:



CRM меняет статус



После завершения заказа:



через 24 часа:



создается задача:



"Запросить отзыв клиента"
