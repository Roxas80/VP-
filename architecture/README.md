# Архитектура решения

## Общая схема

```
┌─────────────┐     ┌─────────┐     ┌─────────────────┐
│   Клиент    │────→│ Telegram│────→│   Telegram Bot  │
│  (Instagram)│     │   Bot   │     │  (Python/Node)  │
└─────────────┘     └─────────┘     └────────┬────────┘
                                             │
┌─────────────┐     ┌─────────┐              │
│   Клиент    │────→│WhatsApp │──────────────┤
│  (Instagram)│     │Business │              │
└─────────────┘     └─────────┘              ↓
                                      ┌─────────┐
                                      │  Бро    │
                                      │(OpenClaw)│
                                      └────┬────┘
                                           │
                                           ↓
                                      ┌─────────┐
                                      │Оператор │
                                      │(заказы) │
                                      └─────────┘
```

## Компоненты

### 1. Telegram Bot
- **Технология:** Python (python-telegram-bot) или Node.js (node-telegram-bot-api)
- **Хостинг:** Временно — локально или на бесплатном хостинге (Heroku/PythonAnywhere)
- **База данных:** SQLite (для MVP) или JSON-файлы

### 2. WhatsApp Business
- **Приложение:** Стандартное WhatsApp Business
- **Автоматизация:** Быстрые ответы + ярлыки
- **Без API:** Ручная обработка на старте

### 3. Бро (OpenClaw)
- **Роль:** Оркестратор, обновление каталога, аналитика
- **Доступ:** К файлам проекта, логам, документации

## Поток данных

1. Клиент переходит из Instagram
2. Выбирает канал (Telegram/WhatsApp)
3. В Telegram: автоматический заказ
4. В WhatsApp: ручная обработка
5. Заказ поступает оператору
6. Подтверждение → доставка → оплата

## Масштабирование

### Phase 2 (после 50+ заказов)
- WhatsApp Business API
- Платёжная интеграция Kaspi
- CRM (Airtable/Notion/самописная)

### Phase 3 (после 200+ заказов)
- Собственное приложение
- Интеграция служб доставки
- Программа лояльности
